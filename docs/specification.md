# Спецификация требований к ПО (Software Requirements Specification)
**Проект:** Course CLI
**Версия:** 1.0.0

## 1. Назначение системы
**Course CLI** — это консольное приложение (CLI) для преподавателей и методистов, предназначенное для стандартизации процесса создания образовательных курсов. Приложение автоматизирует создание структуры директорий, проверку метаданных и связей между файлами, а также формирует аналитические отчеты с записью событий в формате xAPI.

## 2. Архитектура и стек технологий
*   **Язык программирования:** Python 3.12+
*   **CLI-фреймворк:** `Click` (маршрутизация команд, обработка аргументов).
*   **Обработка конфигураций:** `PyYAML` (парсинг метаданных курса).
*   **Тестирование:** `pytest` (unit- и интеграционные тесты).
*   **Контейнеризация:** `Docker`.
*   **Дистрибуция:** Опубликовано в глобальном реестре `PyPI`.

## 3. Спецификация данных (Data Models)

### 3.1. Структура `course.yaml` (Манифест курса)
Файл является единым источником истины (SSOT) для курса.
*   `course_name` (string): Название курса.
*   `modules_count` (int): Количество модулей (обязательно > 0).
*   `lessons_per_module` (int): Количество уроков в модуле.
*   `outcomes` (list[string]): Ожидаемые учебные результаты. Должен содержать минимум 1 элемент.
*   `skills` (list[string]): Формируемые компетенции (опционально, но рекомендуется).

### 3.2. Спецификация событий xAPI
Логирование производится локально в файл `log.json` согласно профилю xAPI (Experience API). 
Структура объекта (JSON):
*   `actor`: Определен системно (name: "Course CLI User", mbox: "mailto:cli-user@example.com").
*   `verb`: Идентификатор действия (`http://adlnet.gov/expapi/verbs/initialized`, `validated`, `reported`).
*   `object`: Идентификатор курса как объекта (`http://example.com/courses/{course_name}`).
*   `timestamp`: Временная метка в формате ISO 8601.

## 4. Функциональные требования

| Команда | Описание | Ожидаемый результат | Код возврата (Exit Code) |
| :--- | :--- | :--- | :--- |
| `init <name>` | Инициализация нового курса (Scaffolding). | Создана структура папок `modules/`, сгенерированы `.md` файлы и `course.yaml`. Записан лог xAPI. | `0` (Успех) |
| `validate <name>` | Проверка структуры курса и целостности связей. | Проверено наличие обязательных ключей в YAML. Найдены файлы, указанные в гиперссылках (RegEx парсинг). Если есть битые ссылки или пустые массивы — вывод ошибок. Записан лог xAPI. | `0` (Успех), `1` (Ошибки валидации) |
| `report <name>` | Генерация аналитического дашборда. | Вывод количества модулей, уроков, метрик заполненности контента. Подтверждение пользователя для локального сохранения xAPI-отчета. | `0` (Успех) |

## 5. Нефункциональные требования
*   **Автономность:** Инструмент должен работать локально, без обязательного подключения к интернету или внешним LRS серверам.
*   **Производительность:** Время валидации курса размером до 100 Markdown-файлов не должно превышать 2 секунд.
*   **Изоляция:** Приложение должно быть способно работать в контейнере Docker без установки зависимостей на хост-машину.
